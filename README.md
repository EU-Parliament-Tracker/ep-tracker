# EP Tracker

An open-source tool to track European Parliament activity вЂ” MEP profiles,
voting records, plenary sessions, legislative documents, and parliamentary
questions вЂ” updated daily from the official EP Open Data API.

Built with **Jekyll + Minimal Mistakes** and hosted on **GitHub Pages**.
No database, no backend: just JSON files committed to git and a static site.

---

## Quick start

### 1. Clone and install

```bash
git clone https://github.com/your-org/ep-tracker.git
cd ep-tracker
bundle install
pip install -r fetch/requirements.txt
```

### 2. Fetch data (first run)

```bash
python fetch/fetch_ep_data.py
```

This writes all JSON files into `_data/`. Expect ~5вЂ“10 minutes on first run
(one API call per MEP). Subsequent daily runs are much faster since only
changed data is re-fetched.

### 3. Serve locally

```bash
bundle exec jekyll serve
# в†’ http://localhost:4000
```

## Deployment (GitHub Pages)

1. Push the repo to GitHub
2. Go to **Settings в†’ Pages в†’ Source** and set branch to `gh-pages`
3. The GitHub Actions workflow runs automatically every day at 06:30 UTC
4. To trigger a manual refresh: **Actions в†’ Fetch EP data & deploy в†’ Run workflow**

Update `_config.yml` with your repo's `url` and `baseurl` before first deploy.

---

## Customising

- **Add a data source**: write a `fetch_<thing>()` function in `fetch_ep_data.py`
  and call it from `main()`. The output JSON is immediately available in
  Liquid templates as `site.data.<thing>`.
- **Change the update schedule**: edit the `cron` line in `.github/workflows/fetch-data.yml`
- **Change the lookback window**: adjust `LOOKBACK_DAYS` in `fetch_ep_data.py`
- **Theme skin**: change `minimal_mistakes_skin` in `_config.yml`
  (options: default, air, aqua, contrast, dark, dirt, mint, neon, plum, sunrise)
